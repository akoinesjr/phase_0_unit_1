GPS 1.1: First Pairing Session

Pair Programming

People often think of programmers as working primarily alone, but recently developers have taken to working in pairs. This "pair programming" approach allows developers to learn from each other and work more efficiently.

To pair program effectively, it is important that both participants communicate with one another and have clearly defined roles.

Typically, these roles are the "driver" and the "navigator".

Driver

  • Uses the keyboard
  • Takes direction from the navigator

Navigator

  • Tells the driver what to write
  • Directs the overall though process in the exercise

Reflection

Building this page together was a good introduction to the pair programming experience. We struggled a bit initially with the CSS, but were able to achieve a result we were happy with through effective communication.
